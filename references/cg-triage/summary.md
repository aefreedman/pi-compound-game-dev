# Summary

## Summary Template

```
Triage complete

Total items: X
Approved (ready): Y
Skipped (pending): S
Deleted: D

Approved:
- <file> - <short description>

Skipped:
- <file> - <reason>

Deleted:
- <file> - <reason>

Status constraints:
- P1 ready items block merge/release until resolved or explicitly accepted.
- P2 ready items should be handled before shipping when practical.
```

Do not add generic next-step menus. If a context-specific next action is already known from the user's request or workflow state, mention it briefly; otherwise stop at the summary.
